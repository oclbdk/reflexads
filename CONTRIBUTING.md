# Contributing

This repo is meant to be forked. Running your own copy is the fastest way to
understand how it's run, and every fork is its own instance: your record,
your standards, your public site. The live walkthrough is at `/contribute`
on any instance's site.

## Try it

1. Fork and clone.
2. Open the checkout in an AI harness and get the dev server running
   (`cd site && npm install && npm run dev`).
3. Work locally. Commit when something feels done. A good first task:
   update the Contributor's Guide to match your fork.
4. Turn on GitHub Pages, once: Settings → Pages → Source: GitHub Actions.
5. Push. Your fork is live at `https://<you>.github.io/<repo>`.

## Exchange

- PRs against `main` are welcome. Review is ad hoc: a PR might get merged,
  reworked, or never answered. Your copy is already live either way.
- Taken patches land as merge commits, authors and messages intact.
- Provenance rides in the commits: model hands as `Co-Authored-By` trailers.
  Generated files stay uncommitted.
- Pull back whatever you want, span by span. Expect breaking changes with no
  migration notes.
